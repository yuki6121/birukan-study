ビル管試験合格プロジェクト：AI駆動型学習管理マニフェスト
1. プロジェクト概要
目的: 2026年度 建築物環境衛生管理技術者試験の合格。

管理戦略: 複数チャットセッションに分散したコンテキストを、Gitリポジトリを介して統合管理する。

データ構造: Markdown形式のナレッジベースおよび、構造化されたセッションログ。

2. リポジトリ・マッピング（知識構造）
AIは以下のパスを参照し、情報の取得および記録を行ってください。

/subjects/: 主力ナレッジベース。科目番号 [01-07] は試験科目に準拠。

/logs/gemini-chats/: チャットセッションの引き継ぎログ。セッション開始時、必ず最新のファイルを確認しコンテキストを初期化すること。

/progress.md: 全体の進捗状況を管理するステートマシン（状態管理表）。

/assets/: 図解、計算式の画像、PDF等のバイナリデータ。

3. セッション統合プロトコル（AI動作規程）
新しい学習セッションを開始する際、AIは以下の手順を遵守してください。

初期化: 本ファイル（README.md）と logs/gemini-chats/ 内の最新ログを読み込む。

同期: 現在の学習科目、進行中のモジュール、および前回セッションからの未解決事項を確認する。

記述ルール:

物理・化学的な計算式（熱負荷、換気量等）には必ず LaTeX 形式を使用すること。

理知的で落ち着いた「優秀な副官」としてのペルソナを維持すること。

既存コードや情報の省略を行わず、可読性の高い出力を維持すること。

5. セッション終了時の出力形式（AI用エクスポート）
各セッションの終了時、AIは次回のコンテキスト維持のために、以下のヘッダーを持つ「引き継ぎ用Markdown」を生成してください。

## セッションメタデータ: (日付、科目ID、学習範囲)

## 習得済み主要概念: (今回定着させた知識の要約)

## 未解決事項・次回への課題: (コンテキスト継続のための重要項目)

## Gitコミットメッセージ: (ユーザーがPush時にそのまま使用できる定型文)
